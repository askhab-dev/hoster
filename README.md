# Hoster

Проект представляет собой веб-приложение на основе React 19 + TypeScript, собранное с помощью Vite.


## 🛠 Установка и запуск

Требования:
- Node.js 18+
- npm или другой установщик пакетов

```bash
# установить зависимости
npm i

# запуск в режиме разработки
npm run dev

# проверка линтинга/кодстайла
npm run lint
npm run format

# полный билд для продакшна
npm run build
# предварительный просмотр собранного приложения
npm run preview
```

## 🧱 Структура

Структура проекта выстроена по концепции **Feature Sliced Design (FSD)**:
- `app` – старт приложения, глобальная конфигурация, роуты.
- `pages` – полноценные страницы (каждая страница – отдельная фича со своими компонентами, стилями, config).
- `widgets` – переиспользуемые UI блоки, которые могут встретиться на разных страницах.
- `shared` – общий UI, утилиты, типы, хелперы, стили; самый низкий уровень, не зависит от фич.
Каждая папка внутри `pages` или `widgets` представляет собой конкретную функциональную единицу, содержащую собственные субмодули (`ui`, `assets`, `config`).

Дополнительные правила:

1. **Типизация** – строгий режим (`strict: true` в `tsconfig`), избегать `any`.
2. **SCSS-модули** – стили находятся рядом с компонентами и импортируются как модули.
3. **Ленивая загрузка** – используются `React.lazy`/`Suspense` или динамический `import` для разделения кода.
4. **Роутинг** – маршруты определяются в `app` и загружают страницы лениво.
5. **Конфиги** – статические данные (баннеры, FAQ, разделы) хранятся в папках `config` внутри фич.
